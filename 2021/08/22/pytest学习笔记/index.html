August 22, 2021 3735

安装

pip install pytest

基本概念

测试通过

用 . 标识Passed

测试失败

用 F 标识 Failed

断言

pytest使用assert断言实际结果和预期结果是否相同

异常捕获

有时需要测试程序能否正常抛出异常, 此时pytest需要将该异常捕获

使用方式

函数命名方式

类似unitest, pytest也会自动去查找指定命名方式的函数, 自动将这些函数作为测试目标

函数要以test_开头

执行命令

  • 执行指定文件下所有函数

  • 指定制定的函数

  • 模糊匹配函数

函数标记

  • @pytest.mark.标记名

指定执行的标记名

支持指定多个标记

跳过测试

有些case暂时不需要跑, 可以打上跳过标记

  • @pytest.mark.skip(reason=“跳过测试的理由”)

测试结果中 s表示测试被跳过

  • @pytest.mark.skipif(条件)
    符合条件的就跳过测试

预期错误

  • @pytest.mark.xfail(条件语句, 原因)

测试结果中 x 表示预期的失败

参数化

测试用例有时需要测试很多情况下的错误, 不同的数据就可以来模拟多种情况. 比如要有常规数值\临界数值\越界数值

  • @pytest.mark.parametrize(‘参数名’,[参数列表])
    在函数中就可以使用该参数, pytest根据参数列表长度自动执行多次
  • @pytest.param
    可以在执行过程中展示详细的信息

固件

固件就是函数. pytest会在执行前或者执行后 运行这些函数

  • 使用@pytest.fixture()定义固件;
  • 可以把所有的固件都写在conftest.py中, pytest会自动加载

执行前后的函数定义方式和使用方式

固件的作用域

​ @pytest.ficture(scope=‘作用域’)

​ 作用域有如下:

  • function: 函数级，每个测试函数都会执行一次固件； 默认作用域
  • class: 类级别，每个测试类执行一次，所有方法都可以使用；
    要是用@pytest.mark.usefixtures(‘固件方法名’)
  • module: 模块级，每个模块执行一次，模块内函数和方法都可使用；
  • session: 会话级，一次测试只执行一次，所有被找到的函数和方法都可用。

monkeypatch

运行时动态修改类或者模块

  • setattr(target, name, value, raising=True)，设置属性；
  • delattr(target, name, raising=True)，删除属性；
  • setitem(dic, name, value)，字典添加元素；
  • delitem(dic, name, raising=True)，字典删除元素；
  • setenv(name, value, prepend=None)，设置环境变量；
  • delenv(name, raising=True)，删除环境变量；
  • syspath_prepend(path)，添加系统路径；
  • chdir(path)，切换目录。

修改对象的函数

rewarn
